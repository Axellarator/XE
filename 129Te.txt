     2.Additional function.                                        //~@@@1I~
       -Syntax Highlight.                                          //~@@@1R~
		Supports ANSI:reverse attributes of Andre Simon's Highlight and GNU source-highlight.//~@@@1I~
		Also, by specifying C_UNDERLINE=R/G_UNDERLINE=R in xesynl.cfg/::xesynl.cfg,//~@@@1I~
        you can get reverse effect for underline attribute.        //~@@@1I~
	   -Unicode support for CID prefix.                            //~@@@1I~
	    U-235d(‚çù) is a comment prefix in APL language, CID supports it.//~@@@1I~
		You can also set the extensions to apply by specifying the following in the ::xe.ini file://~@@@1I~
          Change_ID_Type_4  = 60 = u-235d= APL                     //~@@@1R~
		With this setting, you can specify APL by CID type like as "CID on APL"//~@@@1I~
        When using Unicode, the display width varies depending on the language environment,//+@@@1I~
        so be careful when sharing the file.                       //+@@@1I~
        For example, if Windows displays by 2 columns and Linux displays 1 column,//+@@@1I~
        the CID tagged to the update line on Windows is not recognized as CID on Linux.//+@@@1I~
        So the "cid clear" command on Linux may return 0 rows.     //+@@@1I~
